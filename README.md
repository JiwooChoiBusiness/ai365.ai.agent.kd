# NAVIEN AI AGENT — 지식 수집·적재 에이전트 (데모)

[![tests](https://github.com/JiwooChoiBusiness/ai365.ai.agent.kd/actions/workflows/test.yml/badge.svg)](https://github.com/JiwooChoiBusiness/ai365.ai.agent.kd/actions/workflows/test.yml)

AI가 외부 자료를 수집해 **저작권 리스크를 판정**하고, 검증을 통과한 문서만 **SharePoint(사내 지식 저장소)에 비가역 적재**하는 파이프라인의 **인터랙티브 데모**입니다.

**▶ 라이브 데모: https://jiwoochoibusiness.github.io/ai365.ai.agent.kd/**

> ⚠️ **이 저장소는 순수 프런트엔드 데모입니다.** 백엔드·실제 API 연동은 없으며, 모든 데이터는 가상의 시드이고 모든 동작(수집·판정·업로드)은 브라우저 안에서 시뮬레이션됩니다. 등장하는 문서·계약·이메일·URL은 전부 예시입니다.

## 핵심 설계 원칙

| 원칙 | 의미 |
|---|---|
| **업로드는 비가역** | 적재된 문서는 회수·삭제 기능이 존재하지 않음 — 안전장치는 사전 게이트(원본 대조·업로드 대기)와 사후 조치(인시던트 대장) |
| **5레벨 판정** | L1 자유 / L2 조건부 / L3 불명(보호가 기본값) / L4 명시 제한 / L5 접근통제(수집 차단) |
| **표준 카탈로그 우선** | 라이선스 표기를 코드 매칭으로 먼저 판정(32항목, L1~L5 전 레벨), AI 정밀 분석은 카탈로그 미스만 |
| **레벨 상향은 라이선스로만** | 이용 가능 = 기본 레벨 ⊕ 권리 원장의 유효 계약 ⊕ 조직 정책 — 애매하면 항상 보수적으로 |
| **동시 배포, 분리 불가** | SharePoint 적재 순간 사내 AI 검색(RAG)·교육 플랫폼(Viva)에 함께 전파 — 그래서 적재 자격은 3용도(열람·AI·교육) 전부 충족 필요 |

## 화면 구성

**검토 대기**(우선순위 정렬·일괄 승인·키보드 J/K/A/H/R) · **수집 로그** · **업로드 대기**(취소 가능한 마지막 구간) · **적재함** · **보류·차단** · **권리 원장**(계약 + 표준 카탈로그) · **사후 조치** · **감사 로그**(필터·검색), 그리고 설정(정책 드라이런 미리보기·수집 소스·표기 키워드·분류/폴더/파일명·연동)과 알림 드로어.

## 실행

```bash
# 그냥 열기 — 빌드 불필요
mockup/knowledge-agent-v2.html 를 브라우저로 열기

# 회귀 테스트 (jsdom, 49 assertions)
npm install
npm test
```

## 기술

의존성 없는 **단일 HTML 파일**(순수 HTML+CSS+JS). 상태 머신·룰 테이블(R01~R15)·표준 카탈로그·판정 엔진이 전부 파일 안에 있으며, `[BACKEND]` 주석이 실제 서비스에서 API로 대체될 지점을 표시합니다.
